## 一、架构总览

- **【神圣规划】**
  - [**向导·派蒙**](paimon/paimon.md)（统一入口）
    - 入口接入：统一对接 WebUI / Telegram / QQ 等渠道
    - 轻鉴权：各 channel 沿用原有机制（WEBUI_ACCESS_CODE / BOT_TOKEN+OWNER_ID / APPID+SECRET）
    - 轻量安全：关键词过滤、恶意参数拦截
    - 意图粗分类：判断任务类型 + 复杂度
    - 任务路由：简单 → 天使；复杂 → 四影-七神
    - 天使调度：按 skill 派发、30s 超时、失败走魔女会流转
    - 闲聊响应：走浅层 LLM，不经任何天使
    - 权限询问：启动读世界树画像；运行时只接四影通知；敏感操作询问用户；识别"永久"写世界树
    - 指令规则：「/task」开头强制路由四影-七神
  - **【第一轨】**[**天使体系**](angels/angels.md)（简单任务）
    - 天使 = skill 代名词，一一对应
    - 1~2 个天使能完成 → 派蒙直调
    - 无法处理 / 实际复杂 → 魔女会 → 生执重新编排
    - 内置 30s 超时
  - **【第二轨】四影-七神**（复杂任务）
    - **四影**（流程骨架，不做业务）
      - [**生执·纳贝里士**](shades/naberius.md)：任务编排、DAG 拆分、依赖环检测、多轮迭代轮次控制、失败回滚
      - [**死执·若纳瓦**](shades/jonova.md)：安全审查（违规 / 越权）+ 规则合规校验 + 运行时新 skill 审查
      - [**空执·阿斯莫代**](shades/asmoday.md)：动态路由、服务发现、故障切换
      - [**时执·伊斯塔露**](shades/istaroth.md)：活跃压缩 + 生命周期 + 最终归档 + 最终审计
    - **七神**（能力模块）
      - [**风神·巴巴托斯**](archons/venti.md)：自由·歌咏 → 时事新闻采集 + 新闻推送整理（✅ 信息流面板）
      - [**岩神·摩拉克斯**](archons/zhongli.md)：契约·财富 → 理财（红利股 / 资产 / 退休规划）+ 股价 / 分红提醒整理（✅ 理财面板）
      - [**雷神·巴尔泽布**](archons/raiden.md)：永恒·造物 → 写代码（含自检）
      - [**草神·纳西妲**](archons/nahida.md)：智慧·文书 → 推理、意图、知识整合、文书起草、Prompt 调优、个人偏好管理（✅ 知识 / 偏好面板）
      - [**水神·芙宁娜**](archons/furina.md)：戏剧·评审 → 游戏 + 成品评审（✅ 游戏面板）
      - [**火神·玛薇卡**](archons/mavuika.md)：战争·冲锋 → shell/code 执行、重型工具、技术性重试
      - [**冰神·冰之女皇**](archons/tsar.md)：反抗·联合 → skill 生态全管（发现 + 写世界树 + AI 自举）（✅ 插件面板）
  - **全局支撑层**
    - [**三月女神**](foundation/march.md)：守护进程、任务观测（✅ 观测面板）、定时调度、推送响铃（定时 + 事件）

## 二、交互流程

### 2.1 请求流（用户 → 系统）

```text
用户 → channel（WebUI / TG / QQ）→ 派蒙
  （派蒙做轻鉴权 + 轻量安全 + 意图粗分类）
   │
   ├── 闲聊 / 简单问候
   │     └── 派蒙自己走浅层 LLM 回复（不经天使、不经四影）
   │
   ├── 简单任务（1~2 个 skill 能完成）
   │     └── 派蒙直调天使
   │           （查本地权限画像，敏感操作询问用户；天使失败则兜底走魔女会）
   │
   └── 复杂任务 / 「/task」强制指令
         └── 派蒙直接进入四影
               → 死执（安全审查）→ 生执（DAG 编排）→ 空执（路由）→ 七神执行
```

**两种进四影的路径**：
| 路径 | 触发 |
|---|---|
| 派蒙直送 | 派蒙自动判定为复杂任务，或用户「/task」强制指令 |
| 魔女会兜底 | 天使处理失败 / 天使发现任务实际复杂 / 派蒙判定超出天使能力 |

魔女会通道只是"**天使 → 四影**"的兜底桥，不是用户请求进四影的主路径。

### 2.2 响应流（系统 → 用户）

```text
天使路径:   天使结果 ───────────────→ 派蒙 → channel → 用户
四影路径:   七神产物 → 时执收尾 → 派蒙 → channel → 用户
三月提醒:   定时任务触发 → 三月 → 派蒙 → channel → 用户
```

**派蒙是唯一出入口**——三月 / 七神 / 死执等都不直接调 channel；派蒙挂掉时三月只做拉起 + 暂存，绝不代发。

### 2.3 典型复杂任务流

以"帮我写一个 XX 功能"为例：

```text
用户 → channel → 派蒙
  派蒙：轻鉴权 + 轻量安全 + 意图粗分类 → 判定为复杂任务
   │
   ▼
【进入四影】
死执：深度安全审查（违规 / 越权 / 合规）
   │
   ▼
生执：DAG 编排（拆子任务 + 静态依赖环检测）
   │
   ▼
空执：把子任务路由到对应七神
   │
   ▼
【七神能力层·多轮迭代（轮次由生执控制）】
   草神（思考 + 写方案）
     → 水神（评审方案；不通过则回草神改）
     → 雷神（写代码 + 自检）
     → 水神（挑刺代码；不通过则回雷神改）
     → ...（必要时草水雷多轮循环，直到水神通过）
   （期间若涉及敏感操作，死执批量询问用户一次）
   ※ 火神仅在用户明确要求"执行 / 部署 / 运行"时才出现；本例仅写代码，不触发火神。
   │
   ▼
时执：最终归档 + 审计复盘
   │
   ▼
派蒙：收到时执交接的最终产物 → 人格包装 → channel → 用户
```

**分工提醒**：
- 派蒙只负责**入口接收 + 出口送达**，中间不干预
- 四影负责**流程骨架**（审查 / 编排 / 路由 / 收尾）
- 七神只负责**执行自己的业务域**，不做编排

整体调用顺序固定：**死执 → 生执 → 空执 → 七神 → 时执**（前三为四影前置、七神为业务执行、时执为四影收尾）。

### 2.4 权限询问流程

**天使路径（派蒙单项询问）**：
```text
调用天使前 → 派蒙查本地缓存
  ├── 永久放行 → 直调 + 提示"按之前的永久授权放行"
  ├── 永久禁止 → 拒绝 + 说明
  ├── 普通权限 → 放行 + 友好告知"这次我要用下 X 能力哦~"
  └── 敏感权限无记录 → 询问用户，按答复处理
```

**四影路径（死执批量询问）**：
```text
生执编排 DAG → 死执扫描 DAG 敏感操作 → 排除已永久放行的项
  → 剩余一次性打包询问用户（全同意 / 逐项 / 拒绝）
```

**永久授权关键词**：用户必须明确说"永久 / 以后都..."才入库；只说"放行 / 同意 / 拒绝"仅本次有效。

### 2.5 新 skill 上线流程

```text
冰神（扫 skills/ 目录 / 监听外部 plugin / AI 自生成）
  ├── 启动时扫的内置 skill → 直接写世界树（代码审查已把关，跳过死执）
  └── 运行时新增（plugin / AI 生成）
      → 魔女会 → 死执审查权限声明
         ├── 通过 → 冰神写世界树 + 四影通知派蒙更新缓存
         └── 拒绝 → 冰神拒绝装载 + 告知用户原因
```

冰神是 skill 生态的**唯一写入者**；派蒙只从世界树读，不扫 skills 目录。

### 2.6 推送流程（三段式 + 两种触发）

推送链路由**三个角色**分工：

| 角色 | 职责 |
|---|---|
| **数据收集者**（风神 / 岩神 / 其他） | 收集数据 → 存储 → 整理要推送的内容 |
| **三月** | 响铃：决定"什么时候该推" |
| **派蒙** | 送达：拿到整理好的内容 → 按人格包装 → 经 channel 送达用户 |

**两种触发方式**：

```text
定时触发:
  三月定时器(到点) → 通知数据收集者整理内容 → 派蒙 → channel → 用户

事件触发:
  数据收集者(感知到重要数据 + 整理好内容) → 请求三月响铃 → 派蒙 → channel → 用户
```

**规则**：
- 三月不直发；派蒙挂掉时三月保留积压提醒，派蒙恢复后由派蒙补发
- 数据收集者不直接找派蒙，必经三月响铃（保持三月是调度中枢）
- 具体推送时机 / UX / 打断策略等细节本轮不展开，后续再定

### 2.7 永不绕过派蒙（铁律）

- 所有对用户的输出必须经派蒙（统一人格 / 格式化 / 打断判断）
- 三月、七神、死执等都不直接调 channel
- channel 是派蒙独占的出入口
