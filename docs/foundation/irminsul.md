# 世界树

> 隶属：[神圣规划](../aimon.md) / 基础层
> 相关：[权限与契约](../permissions.md)

**定位**：提瓦特世界树为原型，承担**知识 + 缓存 + skill 生态登记 + 用户授权**的核心存储枢纽。

## 核心能力

- **知识库存储**：原有 `knowledge/` 目录的持久化层
- **缓存层**：热数据快速访问；由三月定时触发预热
- **skill 生态登记**（权威源）：
  - 所有 skill 声明（包括内置从 `skills/` 扫进来的 + 冰神动态装载的 plugin + 未来 AI 生成的）
  - 由**冰神**唯一写入
- **用户授权记录**：永久放行 / 永久禁止清单（写入场景详见下方"访问接口与权限"）
- **只存不推**：不做事件广播；消费者的感知机制详见各自文档

## 与草神的边界

- 世界树 = 底层存储（原始知识、个人偏好、用户授权、skill 生态声明的落盘）
- 草神 = 在世界树之上做推理、整合与**业务接口**（面板层的所有读写操作由草神承接，最终存取走世界树）

## 访问接口与权限

世界树只做存储，提供 **读 / 写** 两类接口，并对"写"做严格限权（缓存层为内部加速，不对外暴露读写）：

| 数据域 | 可读 | 可写（仅允许以下来源） |
|---|---|---|
| 知识库 | 草神（业务接口收口） | 草神（业务接口收口） |
| skill 生态声明 | 派蒙 / 死执（启动时） | 冰神（唯一写入者） |
| 用户授权记录 | 派蒙 / 死执（启动时）、草神面板（UI 展示时） | 派蒙（对话写）、草神面板（撤销写） |

> 具体"谁什么时候来读 / 写、为什么"详见对应模块的职能文档和 [权限与契约](../permissions.md)。
