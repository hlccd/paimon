# 权限与契约体系（跨模块协作机制）

> 隶属：[神圣规划](aimon.md)
> 相关：[派蒙](paimon/paimon.md) · [死执](shades/jonova.md) · [世界树](foundation/irminsul.md) · [冰神](archons/tsar.md) · [草神](archons/nahida.md)

跨越派蒙 / 四影 / 七神 / 世界树的统一权限机制，保证天使体系和四影体系都有安全防线。

## 核心角色分工

| 角色 | 职责 | 比喻 |
|---|---|---|
| **世界树** | 全系统**唯一存储层**；权限相关的两个数据域：skill 生态声明 + 用户授权记录（只存不推） | 档案总库 |
| **冰神** | skill 生态管理者：扫目录 / 监听外部 / AI 自举；走四影审查；唯一写世界树 | 生态管理员 |
| **派蒙** | 启动时从世界树读 → 本地缓存；运行时接四影通知；天使路径权限询问；用户永久授权时写世界树 | 一线执行者 |
| **死执** | 启动时从世界树读；复杂任务批量扫描 + 确认；新 skill / plugin 上线审查 | 主审法官 |
| **四影** | 新 skill 上线的审查通道；审查通过后通知派蒙更新缓存 | 海关 |
| **草神面板** | 提供"查看永久授权 + 撤销永久授权"的 UI 入口；读写都走世界树 | 档案阅览窗口 |

## 敏感度分级

| 等级 | 定义 | 举例 |
|---|---|---|
| **普通** | 纯读 / 本地无副作用 / 低风险 | `web_search`、wiki 查询、知识库读 |
| **敏感** | 系统 / 外部 / 凭据 / 可导致副作用的写 | `exec` shell、账号登录、文件系统、未授权抓取、知识库写、定时任务注册 |

## 启动流程

```text
1. 世界树启动
2. 冰神启动：
   a. 扫 skills/ 目录 → 同步声明到世界树（幂等写入）
   b. 从世界树加载历史 plugin 声明
3. 派蒙 / 死执 启动：
   a. 从世界树一次性读 skill 声明 + 用户授权记录 → 本地缓存
   b. 等待四影运行时通知
```

## 运行时对世界树的访问原则

- **被缓存的消费者**（派蒙 / 死执）：不再轮询世界树，只走启动读 + 四影通知两种感知方式
- **用户授权记录的运行时访问**：派蒙对话写 / 草神面板读取展示 / 草神面板撤销写

## 运行时决策

运行时的权限决策详细流程（天使路径 / 四影路径）已在 **[交互流程 2.4 权限询问流程](aimon.md#24-权限询问流程)** 展开，本文不重复。

**核心约束**：
- **缓存优先**：两条路径都只查本地缓存，不直接读世界树
- **分工**：天使路径由派蒙单项询问；四影路径由死执批量询问
- **去重**：同一请求不会被问两次

## 用户答复识别（派蒙铁律）

| 用户说法 | 处理 | 是否入库 |
|---|---|---|
| 放行 / 同意 / 允许 / OK / 好的 | 本次放行 | ❌ |
| 拒绝 / 不要 / 不行 / 不 | 本次拒绝 | ❌ |
| **永久放行** / **永久允许** / **以后都允许** | 放行 + 写世界树 + 自更新缓存 | ✅ |
| **永久禁止** / **永久拒绝** / **以后都不要** | 拒绝 + 写世界树 + 自更新缓存 | ✅ |

**识别原则**：只有当用户明确说"永久 / 以后都..."这类持久性副词时，才触发入库。否则默认仅本次有效。

## 用户体验保障

- **不重复打断**：天使路径派蒙问一次；四影路径死执批量问一次
- **透明提示**：放行时告知用户用了什么能力；走永久记录时说明"按之前的授权放行"
- **可撤回**：用户可通过草神的"知识 / 偏好面板"查看并撤销永久授权记录

## 画像更新链路

派蒙感知画像变化**只有两条路径**，不订阅世界树变更事件。

| 路径 | 触发 | 链路 |
|---|---|---|
| **路径一：内部自更新** | 用户永久放行 / 永久禁止 | 派蒙识别关键词 → 写世界树 → 自更新本地缓存（同模块完成） |
| **路径二：四影通知** | **运行时**冰神装载 plugin / AI 自生成 skill（启动扫入的内置 skill 不走此路径） | 冰神 → 魔女会 → 死执审查 → 通过 → 冰神写世界树 + 四影通知派蒙 → 派蒙更新缓存 |

**关键性质**：
- **只存不推**：世界树不做事件广播
- **不订阅**：派蒙不监听任何世界树变更事件
- **内部闭环**：用户永久授权场景在派蒙内部"写 + 更新"，不跨模块，不会有延迟
- **必过审查**：运行时新 skill 装载不可绕过死执
- **预装免审**：代码提交的 skill 启动时扫入即视为预审通过
