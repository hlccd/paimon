# 时执·伊斯塔露

> 隶属：[神圣规划](../aimon.md) / 第二轨 · 四影
> 整体调用顺序：**死执 → 生执 → 空执 → 七神 → 时执**

**定位**：原初四影之时执，执掌时间与生命周期，对应流程末尾的收尾。

## 核心能力

| 阶段 | 职能 |
|---|---|
| 运行中 | 活跃会话上下文压缩（阈值待定，3.5k 偏低，需调研主流方案） |
| 到期 | 会话生命周期超时管理（复杂任务默认 1 小时） |
| 结束后 · 归档 | 热（30 天）→ 冷（30-90 天）→ 过期自动删除（存储介质待定） |
| 结束后 · 审计 | 流程复盘、异常归因、执行链路审视 |

## 水神评审 vs 时执审计

同样是"审视"，但维度不同：

- **水神 = 内容维度**：方案 / 代码 / 架构本身好不好
- **时执 = 流程维度**：执行链路哪一步出问题、为什么失败
