# 死执·若纳瓦

> 隶属：[神圣规划](../aimon.md) / 四影

**定位**：审生执产出的 skill 草案质量。

> 入口安全审（防恶意 prompt / 工具越权）由派蒙安全模块负责，不归死执。

## 职能

针对每条草案独立审 4 个维度：

1. 内容完整度——步骤可执行、描述不空泛
2. 跟已有 skill 是否重叠（功能冗余直拒）
3. 工具列表是否最小权限
4. 使用边界是否清晰（什么时候用 / 不该用）

裁决三档：

- **通过**：推到用户面板等审批
- **要修**：让生执按反馈重产
- **直拒**：直接归 rejected，不进面板

生执单次最多产出 5 条草案 → 死执循环每条独立审、独立写 verdict。整体严格度取最严档（任一直拒 → 整体直拒）。

生执判定不值得做时死执跳过，不发 LLM。
