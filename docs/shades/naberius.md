# 生执·纳贝里士

> 隶属：[神圣规划](../aimon.md) / 第二轨 · 四影
> 整体调用顺序：**死执 → 生执 → 空执 → 七神 → 时执**

**定位**：原初四影之生执，执掌创造与构建，对应任务编排。

## 核心能力

- **子任务边界划分**：复杂任务拆成独立子任务（DAG）
- **依赖环检测**：静态 + 动态
- **多轮迭代轮次控制**：草神 → 水神 → 雷神 → 水神 循环的轮数上限判定（见 [交互流程 2.3](../aimon.md#23-典型复杂任务流)）
- **失败回滚**（统一承担）：
  - **依赖环回滚**：动态依赖环触发时回滚已执行子任务
  - **执行失败回滚**：子任务执行链路中断时的全链路回滚
  - **共性**：所有回滚场景都会清理中间数据
