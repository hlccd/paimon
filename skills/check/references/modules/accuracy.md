# Module: accuracy（准确性）

## 核心问题

准不准？——声明与现实是否一致。

## 适用输入模式

| 输入模式 | 激活检查项 |
|---------|-----------|
| code | ACC-012 ~ ACC-014（代码注释/docstring） |
| docs | ACC-001 ~ ACC-014, LNK-001 ~ LNK-003 |
| code-vs-docs | ACC-001 ~ ACC-014, LNK-001 ~ LNK-003 全部 |

---

## 检查项

### 文档输入 / 交叉输入

**主检查（ACC，14 项）**：

| ID | 检查项 | 验证方法 | 默认级别 |
|----|--------|---------|---------|
| ACC-001 | 方法名/函数名与源码一致 | Grep 精确匹配定义 | P0 |
| ACC-002 | 参数签名与源码一致 | Read 函数定义，逐参数比对 | P0 |
| ACC-003 | 文件路径真实存在 | Glob 验证路径 | P0 |
| ACC-004 | 行为描述与代码逻辑一致 | Read 实现代码比对描述 | P0 |
| ACC-005 | 配置值与源码一致 | Grep 常量定义 | P1 |
| ACC-006 | API 端点路由与注册一致 | Grep @router 装饰器 | P0 |
| ACC-007 | SQL Schema 与建表语句一致 | Read DB 初始化代码 | P0 |
| ACC-008 | 继承关系与实际 import 一致 | Grep import | P1 |
| ACC-009 | 数值声明与实际一致 | wc -l / Grep count 比对 | P1 |
| ACC-010 | 错误码与源码定义一致 | Grep 错误码常量 | P0 |
| ACC-011 | 核心接口有文档覆盖 | 列出公开方法比对文档 | P0 |
| ACC-012 | 代码注释/docstring 与实际行为不符 | Read 函数实现对照注释 | P1 |
| ACC-013 | CHANGELOG 记录与实际变更不符 | 比对 git log 与 CHANGELOG | P1 |
| ACC-014 | 示例代码输出与实际运行结果不符 | 阅读示例逻辑推演结果 | P0 |

### 代码输入

仅激活 ACC-012 ~ ACC-014（代码内部注释/docstring 准确性）。

## 严重度调整

| 检查项 | 默认 | 本 module | 理由 |
|--------|------|----------|------|
| ACC-005（配置键名拼写错误） | P1 | **P0**（文档输入时） | 配置不会生效 |
| ACC-009（数值偏差 < 20%） | P1 | **P3** | 行数是辅助信息 |

---

## 发现指令

### 视角轮换

1. **签名精确性**：逐项验证方法名、参数名、类型、默认值与源码完全匹配
2. **路径真实性**：每个文件路径通过 Glob 验证
3. **行为一致性**：业务逻辑描述与代码实现比对
4. **数值校验**：配置值、常量、枚举、统计数字与源码一致

### 源码映射策略（交叉模式）

| 文档内容 | 对应源码位置 | 验证方法 |
|---------|-------------|---------|
| 模块概述 | 类定义 | Read 类定义 |
| 方法签名表格 | 函数 def 定义 | Read 精确行号 |
| 配置参数 | 配置加载代码 + 默认值 | Grep 配置键名 |
| 数据库 Schema | 建表语句 / ORM 模型 | Read DB 初始化 |

---

## 验证指令

- 必须交叉验证源码——不能只凭文档内容判断
- 代码注释问题必须读实际函数实现确认行为与注释是否矛盾

---

## 报告侧重

- 每个 finding 附**期望 vs 实际**对比
- 修复建议明确指出文档/注释应改为什么
