# Module: consistency（一致性）

## 核心问题

一不一致？——系统内各处声明/行为是否彼此矛盾。

## 适用输入模式

| 输入模式 | 激活检查项 |
|---------|-----------|
| code | CST-001 ~ CST-005 |
| docs | CON-001 ~ CON-004 |
| code-vs-docs | CON-* + CST-* 全部 |

---

## 检查项

### 文档输入

| ID | 检查项 | 验证方法 | 默认级别 |
|----|--------|---------|---------|
| CON-001 | 同一概念的描述跨文档一致 | 交叉搜索比对 | P1 |
| CON-002 | 数值引用跨文档一致 | Grep 数值 + 上下文 | P1 |
| CON-003 | 架构图与文字描述一致 | 对照图与正文 | P1 |
| CON-004 | 目录索引与实际文件结构一致 | 比对索引与文件列表 | P1 |

### 代码输入

| ID | 检查项 | 验证方法 | 默认级别 |
|----|--------|---------|---------|
| CST-001 | 错误处理模式不一致 | 比对同项目不同模块的 error handling 策略 | P2 |
| CST-002 | API 命名约定不一致 | 检查 camelCase/snake_case/kebab-case 混用 | P2 |
| CST-003 | 配置格式不一致 | 同项目内 YAML/JSON/TOML/env 混用 | P3 |
| CST-004 | 日志格式不一致 | 不同模块日志结构/级别标准不同 | P2 |
| CST-005 | 注释/docstring 风格不一致 | 部分模块详细注释部分无注释 | P3 |

### 交叉输入

CON-* + CST-* 全部激活。额外关注文档中使用的术语与代码中的命名是否一致。

**引用检查**：
- PRJ-007（版本号多处不一致）
- LNG-003（术语不统一）

---

## 发现指令

### 视角轮换

1. **术语视角**：全文检索同一概念的不同称呼
2. **数值视角**：同一数字在不同位置的值是否一致
3. **模式视角**：同一类问题在不同模块的处理方式是否统一

---

## 验证指令

- 不一致不等于错误——可能是有意的差异化设计，需判断是否有正当理由
- 必须给出具体的矛盾对：A 处说 X，B 处说 Y

---

## 报告侧重

- 每个 finding 附**矛盾对**：位置 A 说什么 vs 位置 B 说什么
- 建议统一方向（而非只指出不一致）
