## 发现指令

### 视角轮换

1. **全局引用扫描**：对每个 private 函数/内部类，grep 全项目找调用点
2. **入口点追溯**：从主入口反向 BFS 可达性，不可达的都是候选死代码
3. **条件不可达**：找 `if False`、`if True`、常量条件、始终为空的分支
4. **杂质扫描**：注释掉的代码块、空文件、过时配置项

### 专有关注点

1. 反射/动态调用：`getattr`、`__import__`、`eval`、注册表模式——容易误报
2. 插件发现机制：基于约定命名或元类自动注册
3. 配置驱动执行：根据配置字符串查找函数
4. 测试 fixture：conftest.py、pytest plugins 等间接调用
5. 抽象基类：ABC 本身不是死代码
6. 序列化 schema：pydantic/dataclass 字段即使无显式读取也有效

## 验证指令

- 必须全项目确认——「未使用」结论容易假阳性
- 跨语言调用特别注意——Python 可能被 JS 通过 API 调用
- 导出的 API 不能删——即使项目内无调用
- 测试中的使用也算使用
